- for backup purposes (local copy gets destroyed due to system failure)

Why miight you want to always pull changes manually rather than having Git automatically stay up-to-date with your repository?
- local changes might be overwritten

Describe the differences between forks, clones, and branches. When would you use one instead of the other?
- fork: no permission to work on the original but still give credit to the contributed files
- clone: when you need a copy of the files e.g. on your local computer
- branch: when you want to add feature an keep a working project as backup
